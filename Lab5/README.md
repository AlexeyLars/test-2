# Task Management System

## Описание проекта

Task Management System - это веб-приложение для управления задачами и проектами в командной разработке. Система позволяет создавать проекты, добавлять задачи, назначать исполнителей и отслеживать прогресс выполнения. Приложение поддерживает работу с досками задач в стиле Kanban, фильтрацию и поиск по задачам. Пользователи могут устанавливать приоритеты, сроки выполнения и добавлять комментарии к задачам. Система также включает уведомления о важных событиях и дедлайнах. Проект разрабатывается с использованием современного стека технологий и следует принципам Agile.

## Sprint 1

**Длительность:** 2 недели  
**Цель спринта:** Реализовать базовую функциональность системы управления задачами с возможностью создания, редактирования и просмотра задач.

### Sprint Review

#### Завершённые задачи:
- ✅ Реализована аутентификация пользователей (Issue #1)
- ✅ Создан интерфейс для списка задач (Issue #3)
- ✅ Реализовано создание новых задач (Issue #4)

#### Доставленная функциональность:
Пользователи могут регистрироваться в системе, входить в свой аккаунт, просматривать список своих задач и создавать новые задачи с указанием названия, описания и приоритета.

#### Незавершённые задачи:
- ⏸️ Реализация фильтрации задач (Issue #5) - перенесена в следующий спринт из-за технических сложностей с API
- ⏸️ Добавление комментариев к задачам (Issue #7) - требуется дополнительное время на проектирование

### Sprint Retrospective

#### Что прошло хорошо:
- Команда эффективно распределила задачи в начале спринта
- Ежедневные stand-up встречи помогли быстро решать блокеры
- Базовая функциональность была реализована качественно и с покрытием тестами
- Хорошая коммуникация между frontend и backend разработчиками

#### Что пошло не так:
- Недооценили сложность реализации фильтрации задач (оценили в 2 story points вместо 5)
- Не учли время на code review, что привело к задержкам
- Одна задача (Issue #7) оказалась заблокированной из-за зависимости от внешней библиотеки
- Возникли проблемы с настройкой CI/CD пайплайна

#### Что можно улучшить в следующем спринте:
- Выделять больше времени на детальную оценку сложных задач
- Проводить технический разбор задач перед началом спринта
- Закладывать буферное время на code review (20% от оценки задачи)
- Добавить автоматические уведомления о статусах PR
- Улучшить документацию API для более эффективной интеграции
- Проводить mid-sprint review для корректировки планов
